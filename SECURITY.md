# Security Policy

## Supported Versions

Currently, the `main` branch is the only supported version until formal releases exist.

## Reporting a Vulnerability

Please do not disclose vulnerabilities publicly.

To report a vulnerability, use GitHub security advisories/private reporting if enabled on the repository. Alternatively, contact the maintainer `@rasagyavatsal` through GitHub.

## Scope

Examples of in-scope vulnerabilities include:
- App security issues
- Unsafe file handling
- Dependency vulnerabilities
- Privacy-impacting bugs
